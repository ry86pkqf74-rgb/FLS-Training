# Task5 standard config — sized for v4 (144-sample) run

## Overview

The `finetune_task5_standard.yaml` config was mis-sized for the v4 dataset.
With 144 train samples and effective batch 8, the run is only 18 steps/epoch
× 3 epochs = 54 total optimizer steps — but the config evaluated and saved
every 100 steps, meaning the entire run would have produced **zero evals and
zero checkpoints**. Patched to per-epoch strategy and bumped epochs.

## Context

- v4 dataset materialised at `data/training/2026-04-08_v4` with 144 train /
  27 val / 9 test, 5 task types, sources `critique_consensus / teacher_claude
  / teacher_gpt4o`. v1 had only memorized 25 samples — v4 is the first run
  that's actually past the ≥80 gate.
- The `finetune_task5_standard.yaml` config was inherited from v1/v2/v3 era
  when training was either tiny (memorization runs) or assumed a much larger
  corpus. Nobody had re-sized the eval/save cadence for the in-between regime
  v4 lives in.
- Trainer audit (`grep` against `src/training/finetune_vlm.py`): only the
  flat keys `num_epochs`, `warmup_ratio`, `logging_steps`, `save_strategy`,
  `eval_strategy`, `save_total_limit`, `dataset_path`,
  `resume_from_checkpoint`, `require_vision` are read. Notably **not** read:
  `warmup_steps`, `load_best_model_at_end`, `metric_for_best_model`,
  `greater_is_better`. Any of those in a config would be silently dropped.

## Decision

Patch only the keys the trainer actually honors:

| Key | Before | After | Why |
|---|---|---|---|
| `num_epochs` | 3 | 5 | 54 → 90 optimizer steps; 54 was too thin for a 7B VL LoRA |
| `warmup_ratio` | 0.05 | 0.1 | 0.05 × 90 ≈ 4 steps (rounding-prone); 0.1 → ~9 steps |
| `logging_steps` | 5 | 1 | At ~90 steps total, every step should be visible |
| `save_strategy` | "steps" + `save_steps:100` | "epoch" | Would have saved 0 checkpoints |
| `eval_strategy` | "steps" + `eval_steps:100` | "epoch" | Would have evaluated 0 times |

Deliberately **not** added: `warmup_steps`, `load_best_model_at_end`,
`metric_for_best_model`, `greater_is_better`. These would require a trainer
change in `src/training/finetune_vlm.py`, which is a separate PR.
